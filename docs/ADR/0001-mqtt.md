# ADR 0001: Use MQTT for telemetry transport
## Status
Accepted
## Context
We ingest time-series telemetry from simulated/real devices and need lightweight pub/sub messaging.
## Decision
Use MQTT publish/subscribe for telemetry transport (Mosquitto in dev).
## Consequences
Pros: lightweight, decouples producers/consumers, common in IoT.
Cons: requires topic governance and production-grade auth/TLS.
Mitigations: schema validation at ingress, topic naming conventions, RBAC in API/UI.
